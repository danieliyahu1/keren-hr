KEREN-HR MVP EXECUTION PLAN
============================

Goal
----
Transform the current generic AI chat app (base66) into an HR-focused
recruiting tool called "keren-hr" with minimal changes -- just enough
to reframe the experience, strip away developer-facing features, and
validate the core value proposition with a real user.

Product One-Liner
-----------------
"Paste a job description, get ranked candidates from LinkedIn."

Target User
-----------
HR recruiters and hiring managers at small-to-mid companies who source
candidates manually on LinkedIn. Not technical, but comfortable using
web apps. First user: Atalya (your girlfriend).

CHANGE 8: Agent system prompt -- make it HR-focused
----------------------------------------------------

Files affected:
  - mounts/user1-config/opencode.json
  - mounts/user2-config/opencode.json
  - mounts/atalya-config/opencode.json
  - Java source: UserWorkspaceService.java or wherever the default
    agent prompt is generated for new workspaces

What changes:
  - Current agent prompt:
    "You are the dedicated Base66 agent for user {username}.
     Use all built-in tools when needed..."

  - New agent prompt:
    "You are Keren-HR, an HR recruiting assistant for {username}.
     Your job is to help the user find and evaluate candidates
     on LinkedIn for their open roles.

     When the user describes a role or pastes a job description:
     1. Extract the key requirements (must-haves, nice-to-haves,
        location, seniority level)
     2. Use the LinkedIn skill to search for and evaluate candidates
     3. Present each candidate using the structured format from
        the LinkedIn skill (Name, Title, Verdict, HR Logic,
        Verified Skills, Red Flags)
     4. Be ready to refine the search based on feedback

     Always use the LinkedIn skill's protocols for stealth browsing
     and structured candidate evaluation. Present results clearly.
     Use both global and project skills when relevant.
     Use both global and project MCP integrations when relevant."

What stays: Workspace provisioning infrastructure, agent creation
mechanism, the LinkedIn skill itself, all permission configs.

Effort: 20 min
Risk: Low

==========================================================================
WHAT WE ARE NOT DOING (intentionally)
==========================================================================

- No new screens or layouts
- No candidate cards or structured rendering (free text from agent is fine)
- No database or candidate persistence
- No search history feature
- No changes to backend Java API endpoints (Skills, MCP, Sessions APIs
  stay -- they're just not exposed in the UI)
- No removal of existing API endpoints
- No Java package renaming (too risky for zero user-facing value)
- No test creation (no tests exist currently)
